[size=13]Bonjour à toute la communauté !

Je vous présente aujourd'hui le [b]SimCraft Engine (SCE)[/b], un projet sur lequel je travaille pour transformer RPG Maker MZ en une plateforme de création de jeux plus profonde, plus modulaire et plus puissante. L'objectif est de fournir une base solide pour des jeux de type simulation, gestion et RPG tactique, en s'éloignant des contraintes habituelles du moteur.

Le SCE n'est pas un simple pack de plugins, mais une véritable [b]architecture applicative[/b] qui repense la manière dont les fonctionnalités sont développées et intégrées.[/size]

[center][size=18][u][color=Crimson]La Philosophie du Moteur[/color][/u][/size][/center]

Le cœur du SimCraft Engine repose sur trois piliers fondamentaux :

[quote]
• [b]Modularité[/b] : Chaque fonctionnalité est un module indépendant qui déclare ses propres besoins (dépendances, fichiers de données, etc.). Fini les scripts monolithiques !

• [b]Déclaration[/b] : Un module ne dit pas au moteur [i]comment[/i] faire les choses, mais [i]ce dont il a besoin[/i]. Le moteur se charge ensuite de tout orchestrer.

• [b]Centralisation[/b] : La logique complexe de chargement, de sauvegarde et de débogage est gérée à un seul endroit, le [b]SystemLoader[/b], rendant le système prévisible et facile à maintenir.
[/quote]

Cette approche permet de construire des systèmes complexes de manière propre et organisée, tout en garantissant que les différents modules peuvent cohabiter sans conflits.

• [b]Animations et Poses dynamiques[/b] pour des personnages plus vivants
[/quote]

[center][size=18][u][color=Crimson]Comment ça marche ? Quelques exemples[/color][/u][/size][/center]

[b]Pas besoin d'être un expert en code ![/b]
[size=13]L'un des objectifs du SCE est de rendre des fonctionnalités complexes accessibles à tous. Même si les exemples ci-dessous montrent un peu de code pour les plus curieux, sachez que de nombreux systèmes sont conçus pour être pilotés directement depuis l'éditeur de RPG Maker via des [b]notetags[/b] ou en modifiant des [b]fichiers de configuration[/b] très simples. Vous n'avez pas besoin de savoir programmer pour en profiter ![/size]

[b]Exemple concret : Le système "Paper-Doll" avec les Notetags[/b]
Vous voulez qu'une armure que vous avez créée apparaisse sur votre personnage ? C'est très simple.
1. Dans l'éditeur de RPG Maker MZ, allez dans la base de données, section "Armures".
2. Sélectionnez votre armure (par exemple, une "Cuirasse en plaques").
3. Dans le champ "Note" à droite, copiez-collez la ligne suivante :

[code]
<visualLayer: armor/plate_chest_male 10>
[/code]

[size=13]Et c'est tout ! Quand un personnage équipera cet objet, le moteur ira chercher l'image `plate_chest_male.png` dans votre dossier `img/characters/armor/` et l'affichera automatiquement sur le sprite. Le `10` à la fin indique sa "couche" de priorité, pour s'assurer qu'elle s'affiche bien par-dessus les vêtements mais sous un éventuel tabard.[/size]

[b]Le Cœur du Moteur ([i]System Loader[/i])[/b]
C'est le chef d'orchestre invisible du SCE. Grâce à lui, vous n'avez plus à vous soucier de l'ordre de vos plugins ou de la manière dont ils se chargent. Le moteur gère tout automatiquement. Ajouter une nouvelle fonctionnalité devient aussi simple que de glisser un nouveau module dans le bon dossier. Le `SystemLoader` s'assure que tout fonctionne en harmonie, sans que vous ayez à intervenir.

[b]Des cinématiques puissantes avec des fichiers de configuration[/b]
Fini les centaines de commandes d'événements pour créer une scène ! Avec le SCE, vous décrivez votre cinématique dans un fichier de configuration (au format JSON). Vous définissez les images, les mouvements, les dialogues et les musiques, séquence par séquence.

[size=13]Voici à quoi ressemble un extrait de configuration pour une petite scène :[/size]
[code]
"sequencies": [
    // Étape 1: La musique démarre et un personnage apparaît en fondu
    {
        "audio": { "bgm": { "name": "Theme", "volume": 80 } },
        "layers": {
            "1": { "bitmap": "character1", "opacityGoal": 255, "fadeSpeed": 5 }
        }
    },
    // Étape 2: Un texte s'affiche, la scène attend que le joueur appuie sur "OK"
    {
        "needPressOk": true,
        "storyWindow": { "txt": "Le voyage commence..." }
    }
]
[/code]

[size=13]Comme vous pouvez le voir, même les parties "techniques" sont pensées pour être lisibles et descriptives. Vous décrivez ce que vous voulez, et le moteur s'occupe du comment ![/size]
